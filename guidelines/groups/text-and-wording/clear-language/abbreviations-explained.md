Explanations of :term[abbreviations] are available when first used. 

:::except-when

<p>the abbreviation is:</p>

* used so often it has become a word with its own dictionary entry, such as "scuba," "info," and "HTML"
* used in a logo
* included in a longer phrase, such as "brand DNA," whose meaning needs to be defined to meet the [non-literal language requirement](#non-literal-language-explained). 
:::

:::example
(Pass) ‘He has Avoidant/Restrictive Food Intake Disorder (ARFID).’

(Pass) ‘He has [ARFID].’ — (link to a glossary or tooltip.)

(Fail) ‘He has ARFID.’ — (first reference, no explanation is available.)
:::

:::tests

<b>Procedure</b>

1. Check for any abbreviations in the content.
2. Check that an explanation is available for the first use of the abbreviation.
3. For an abbreviation that has no explanation available, check that it is included in a dictionary intended for the general public.

<b>Expected result</b>
* #2 or #3 is true.
:::
